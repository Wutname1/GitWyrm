# GitWyrm

A fast, focused Git client for Windows. Built with Tauri v2, React, TypeScript, Tailwind CSS v4, shadcn/ui, and a Rust git backend (git2).

## Architecture

```
src/                      React frontend
  layouts/                WorkspaceLayout (tab bar / toolbar / panels / status bar)
  views/                  GraphView (virtualized commit graph), DiffView, SettingsView
  components/ui/          shadcn primitives, skinned to the GitWyrm dark theme
  components/domain/      TabBar, Toolbar, LeftPanel, RightPanel, graph/, diff/, commit-form/
  components/modals/      Onboarding, Clone
  stores/                 Zustand: workspaceStore (repos, recents), uiStore (view state)
  hooks/                  TanStack Query hooks over Tauri commands + file-watcher invalidation
  lib/bindings.ts         GENERATED by tauri-specta — the typed frontend/backend contract

src-tauri/                Rust backend
  src/commands/           repo, log, status, staging, commit, branch, stash, diff, remote
  src/git/graph.rs        Commit-graph lane assignment (unit + integration tested)
  src/git/shell.rs        System git.exe runner (network ops via Git Credential Manager)
  src/watcher.rs          Debounced repo file watcher -> `repo-changed` events
```

Design decisions:

- **Local git ops** use git2 (libgit2) inside `spawn_blocking`; **network ops** (fetch/pull/push/clone) shell out to the system `git.exe` so Git Credential Manager handles all auth.
- **Graph lanes** are computed in Rust during the revwalk; the frontend is a dumb renderer over `@tanstack/react-virtual` (fixed 28px rows, SVG edge overlay).
- **State**: TanStack Query keyed by repo id for all git data; Zustand for UI-only state. The Rust watcher invalidates queries when the repo changes externally.

## Development

Prereqs: Node 22+, Rust stable, VS Build Tools + CMake (for the vendored libgit2), system git.

```bash
npm install
npm run tauri dev      # native window + HMR; regenerates src/lib/bindings.ts
npm run dev            # frontend only in a browser (no git backend)
cd src-tauri && cargo test   # lane algorithm + real-repo + mutation-cycle tests
```

## Release

Builds are NSIS per-user installers (no admin, auto-updating via GitHub releases) plus a raw portable exe (no auto-update).

One-time setup:

1. `npm run tauri signer generate` — put the public key in `src-tauri/tauri.conf.json` (`plugins.updater.pubkey`, currently a placeholder) and add the private key + password as the `TAURI_SIGNING_PRIVATE_KEY` / `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` GitHub Actions secrets.
2. Update the updater endpoint in `tauri.conf.json` if the repo slug differs.

Then: push a `v*` tag. `.github/workflows/release.yml` builds the installer, the `latest.json` updater manifest, and attaches `gitwyrm-portable.exe`.
