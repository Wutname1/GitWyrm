GitWyrm design note · Proposal v2 · independent take

Turn the accent down. Turn the type up.

The current mint #2dd4a7 isn't failing contrast — it's too bright to read comfortably. At ~10:1 against the near-black background, saturated accent text shimmers ("halation"), and it's used as text in 65 places. Meanwhile the type scale bottoms out at 9.5px on a 13px / 400 base — thin and small compound each other. This note treats both together, because they are the same problem: legibility under a dark theme.

Scope — tokens only, no layout changes
Files touched — src/index.css (one block)
Approach — split one accent into two tokens; raise the type floor

01Why "more contrast" made it worse

Dark-theme legibility is a window, not a ladder. Below ~4.5:1 text is muddy; above ~9:1 a saturated hue starts to glow against black and thin strokes vibrate. The mint sits at the glare end, and the app leans on it for links, counts, pills, and hunk headers.

Accent as text, today
9.9 : 1

Contrast of #2dd4a7 on #0b0f14. Passes AAA with room to spare — and that's the problem. The comfortable band for saturated accents on this background is roughly 5.5–8.5 : 1.

Smallest text in the app
9.5px

224 call sites use 9–12px type on a 13px / 400 base. IBM Plex Sans at 400 is already a light-feeling cut; on a dark ground, dark-on-light optics thin it further.

The fix, in one sentence
1 → 2

One accent token is doing two jobs. Fills want a dimmer, deeper value; text wants a calmer, slightly desaturated one. Split them, then raise the base weight to ~450 and the size floor to 11px.

04Type scales

Both app fonts are variable, so intermediate weights like 450 cost nothing. The scales below keep every existing size role — body, secondary, micro — but shift the whole ladder. The mock app above renders with whichever scale is selected.

Floor rule: nothing below 11px, ever. The 9.5–10.5px call sites (branch pills, hints, AI-stage captions) move to the micro token instead of hardcoded pixels — then one variable controls the floor forever.

05Recommendation

Pairing that tested best in the playground above, expressed as the exact token diff.

  • Accent → Deep Mint pair. Fills drop to #1db584 — still unmistakably GitWyrm, but buttons stop shouting. Accent text becomes its own token at #38b78e: ~7.6:1 on the background, inside the comfort band, no shimmer.
  • Type → Comfort scale. Base 13.5px / 450, secondary 12px, micro floor 11px. One weight step and half a pixel — invisible as a change, obvious as an improvement.
  • Migration is mechanical. Existing text-primary usages pick up the new text token automatically once --primary maps to --gw-accent-text; filled components keep --gw-accent. No component edits required for the color half.
  • What stays. Panel ramp, borders, red/amber/green status colors, lane colors — all untouched. This is a two-token, three-number change.
