For the person approving it

Here is what the app actually does.

No floating shield icons. No “enterprise-grade” filler. These are the boundaries a desktop or security lead can review.

Planning a managed rollout?Review the source and release history, then send the team the questions your policy requires.

Install scope

The standard setup installs for the current Windows user. It does not need administrator rights.

Repository access

Local repository operations use the bundled Rust Git library. Fetch, pull, push, and clone run through system git.exe, so Git Credential Manager handles authentication.

Updates

The desktop app checks its public release feed. Update packages are verified against a public key embedded in the app before install.

Diagnostics

Production builds send errors and performance data to Sentry. Default personal data is disabled. Inputs and on-screen text in error replays are masked.

Bring your own AI

GitWyrm does not run an AI service. Connect a provider or GitHub Copilot account you control. Only when you ask for a commit message does GitWyrm send the staged diff and recent commit subjects directly to that provider.

License and source

GitWyrm is published under the MIT License. The source, build workflow, and release history are public on GitHub.

The plain deal

Software should earn trust in specifics.

GitWyrm is made for people doing real work in real repositories. The pitch is short because the source is there when you want the long version.

Every action answers back

Clicks, keys, and drags get a visible result. If work takes time, GitWyrm shows what it is doing.

Plain words beat Git trivia

Choices explain what happens to your files and work. You should not need a glossary to switch a branch safely.

Open when details matter

Read the code. Review the network paths. Use the MIT license. The implementation is part of the product story.

Privacy policy

Proposed website copy · Last updated July 21, 2026

DRAFT FOR LEGAL REVIEW

The short version

GitWyrm works with repositories on your computer. The core app does not require a GitWyrm account. Some features make network requests, and production builds send diagnostic data so crashes can be fixed.

What GitWyrm processes locally

GitWyrm reads repository files, Git history, branches, diffs, and settings needed to show and change your local repositories. Local Git operations are performed on your device.

When data leaves your device

  • Fetch, pull, push, and clone contact the Git remote you chose through system Git and Git Credential Manager.
  • GitHub features contact GitHub after you connect a GitHub account.
  • GitWyrm does not run an AI service. If you connect your own AI provider or GitHub Copilot account and request a commit message, the staged diff and recent commit subjects are sent directly to that provider.
  • Production builds send error, performance, and related diagnostic data to Sentry. Default personal data is disabled. Inputs, media, and visible text are masked in error replays.
  • The app checks the public release feed for updates. This exposes ordinary request data such as IP address and user agent to the hosting provider.

Credentials

Git network credentials are handled by your system Git setup and Git Credential Manager. Credentials for your AI provider and other connected services are stored on your device for the features you configure. GitWyrm does not sell personal information.

Retention and service providers

Local repository data remains until you remove it through the app or your system. Third-party services, including Sentry, GitHub, your Git remote, CDN host, and any AI provider you choose, process data under their own terms and retention rules.

Your choices

You can use the core local Git features without connecting GitHub or an AI provider. You can remove connected providers in settings. You can uninstall GitWyrm and remove its local settings using Windows.

Changes and contact

Material changes will be posted with a new effective date. Until a dedicated privacy contact is published, questions can be opened in the public issue tracker. Do not post secrets or private repository details.
